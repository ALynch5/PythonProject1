# Team Collaboration Log

## Overview

We worked as a group to complete this project using GitHub.

Since this was our first time doing a project like this, we focused on keeping things organised and communicating clearly.



## How we worked as a team

We split the work between us so that everyone contributed.

For example:

* one person focused more on the Flask app
* one worked on the database setup
* one helped with API integration and testing

We also helped each other when someone got stuck.



## Using GitHub

We used GitHub to manage the project.

We:

* created issues for tasks
* used a basic project board (To Do, In Progress, Done)
* worked on separate branches for features
* used pull requests to merge code



## Pull Requests and Reviews

Each feature was added using a pull request.

Before merging:

* another team member reviewed the code
* we checked that it worked properly

This helped us avoid breaking the project.



## Communication

We communicated regularly to keep track of progress.

We discussed:

* what tasks were left
* any problems we were having
* what needed to be fixed

This helped us stay on track.



## Challenges

Some challenges we faced:

* understanding how to connect the database
* working with APIs for the first time
* setting up Docker

We solved these by:

* looking at documentation
* helping each other
* testing things step by step



## What we learned

* how to use GitHub properly in a team
* how to use branches and pull requests
* how to review each other’s code
* how to work together on a shared project



## Conclusion

Overall, we worked well as a team and were able to complete the project successfully.

Using GitHub helped us stay organised and made it easier to collaborate.
